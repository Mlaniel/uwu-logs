# DESIGN.md — uwu-logs Design System

Generated by /plan-design-review on 2026-05-18.
This is the source of truth for all visual decisions in the Vue frontend.

---

## Design Philosophy

Game analytics tool for WotLK raiders. **Dense, intentional, game-native.**

- Functional over decorative. Data density is a feature, not a problem to solve.
- Dark theme is correct for this context — WoW players use dark UIs.
- Game-native colors (class colors, parse rank colors) are sacred. Do not invent alternatives.
- No rounded corners on table rows. No card shadows. No decorative gradients.
- Sort indicators: use `↑` `↓` text characters, not SVG icon libraries.
- Spec filter icons: actual WoW game sprite assets, never HTML styled elements.

### Breakpoints

| Breakpoint | Behavior |
|------------|----------|
| > 768px | Sidebar fixed 220px, main fills rest |
| < 768px | Sidebar becomes hamburger overlay (≡ button top-left of main). Spec filter: `overflow-x: auto` horizontal scroll. |

### Sidebar

- Background: `--surface` (slightly lifted from pure black)
- Boss rows: 36px height, padding 0 12px
- Kill indicator: aquamarine dot or text color
- Wipe indicator: crimson
- Selected boss: `border-left: 3px solid var(--primary)`
- Expand triangle: `▶` collapsed, `▼` expanded (show individual pulls)
- "Show all" toggle: bottom of sidebar, muted style

## Report Page — Spec Filter

- Only render spec icons for **specs present in the current boss segment**
- Icon source: `/static/icons/{spec_icon}.jpg` (actual WoW game spec icons)
- Icon size: 28px × 28px
- Gap between icons: 4px
- Selected: full opacity, `outline: var(--spec-selected-outline)`
- Unselected: `opacity: var(--spec-unselected-opacity)` (0.35)
- Switching bosses: filter resets to "all" automatically

## Report Page — Player Table

### View tabs (above table)

```
  [DAMAGE]  HEAL  TAKEN
```

| Tab | Columns shown |
|-----|--------------|
| DAMAGE | Spec icon, Name (class-colored), Rank, DPS%, Useful Dmg ↓, Total Dmg |
| HEAL | Spec icon, Name, Rank, Heal%, Heal ↓, Total Heal |
| TAKEN | Spec icon, Name, Dmg Taken ↓ |

- Active tab: highlighted with `--primary` underline or background
- Default active tab: DAMAGE

### Sort behavior

- Default sort: Useful Damage descending when DAMAGE tab active
- Active column header shows `↑` (asc) or `↓` (desc) suffix
- `font-variant-numeric: tabular-nums` on all number cells for alignment

### Null-DPS rows (healers in DAMAGE tab)

- Sorted to bottom regardless of active sort key
- Row opacity: `var(--healer-row-opacity)` (0.35)
- DPS% cell shows `—`

### Virtualization

- Use `RecycleScroller` from `vue-virtual-scroller` — raids can have 49+ players
- Do NOT use `v-memo` on PlayerTable rows (RecycleScroller is incompatible)
- Item height: 36px

---

## Component Anti-Patterns (DO NOT)

- No rounded corners on table rows or cells
- No card shadows on any data component
- No colored icon circles (use actual WoW game sprite icons)
- No decorative gradients on backgrounds
- No `system-ui`, `-apple-system`, or `Inter` as font
- No SVG icon libraries for sort indicators — use `↑` `↓` characters
- No `v-memo` inside `RecycleScroller`
